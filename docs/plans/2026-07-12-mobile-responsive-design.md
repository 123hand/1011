# KYC 平台手机端适配设计

## 目标

在不改变企业查询、分析、培训和报告生成逻辑的前提下，为 375px 至 430px 宽度的手机提供完整、可阅读、可触控的业务界面。

## 方案

沿用现有单页 HTML 架构，通过 768px 和 480px 两级 CSS 断点完成响应式重排。桌面端结构保持不变，手机端将顶部导航、搜索区、内容标签、数据卡片、培训模块和报告操作调整为单列或可横向滚动布局。

## 交互约束

- 主要按钮和标签的触控高度不低于 44px。
- 搜索按钮在手机端全宽展示，快捷搜索标签允许横向滚动。
- 内容标签保持单行横向滚动，避免文字压缩和换行造成跳动。
- 表格与图表在自身容器内滚动，不导致页面整体横向溢出。
- 报告导出和打印功能在手机端继续保留。

## 验证范围

- 静态结构测试确认两级断点与关键移动端规则存在。
- 在 375x667、390x844、430x932 视口检查页面无整体横向溢出。
- 运行既有前端、后端与静态资源测试。
- 重新生成 `dist` 和静态部署压缩包。
